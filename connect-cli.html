Access your cluster through the CLI

Target your Resource Group

You need to target a Resource Group (RG) to view your cluster or to add a service (e.g. Cloudant Database). Access to the group and the resources within it are managed by using Identity and Access Management (IAM).

  1. In this lab, the Resource Group hec-iks-group has been created. Select this resource group.

     ibmcloud target -g hec-iks-group

  2. Verify you can see your cluster.

     ibmcloud ks clusters

    Output

  3. Download the Kubernetes configuration files and certificates to connect to your cluster by using kubectl commands

     ibmcloud ks cluster config -c <cluster-name>

  4. The cluster configuration is visible in the folder .kube 

     more .kube/config

  5. You can now execute 'kubectl' commands against your cluster. For example, get the worker nodes of your cluster:

     kubectl get nodes
